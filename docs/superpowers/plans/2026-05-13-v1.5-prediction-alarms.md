# Fit Claw v1.5 Prediction and Alarms Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add personalized performance prediction and threshold alarm tools for agents.

**Architecture:** Keep prediction math, alarm evaluation, and notification delivery in focused domain/lib modules. Tool routes are thin Hono adapters that validate agent input and call deterministic domain functions. Jobs are one-shot scripts so Docker/cron/launchd can schedule refits and daily alarm evaluation without requiring an in-process scheduler.

**Tech Stack:** Bun, TypeScript, Hono, Zod, bun:sqlite, bun:test.

---

## File Structure

- Create `src/domain/predictions.ts`: linear regression fitting, rest-decay tau estimation, cache upsert, `predictPerformance`, `predictSeries`, `refitPredictionModels`.
- Create `src/tools/predictions.ts`: `POST /tools/predict_performance`, `POST /tools/predict_series`, `POST /tools/refit_prediction_models`.
- Create `src/domain/alarms.ts`: alarm rule CRUD, threshold evaluation, notification log writes.
- Create `src/lib/notifications.ts`: agent webhook delivery, Telegram fallback, log-only result metadata.
- Create `src/tools/alarms.ts`: `POST /tools/create_alarm_rule`, `POST /tools/list_alarm_rules`, `POST /tools/disable_alarm_rule`.
- Create `src/jobs/refit-predictions.ts`: one-shot prediction cache refit job.
- Create `src/jobs/evaluate-alarms.ts`: one-shot threshold alarm evaluator job.
- Modify `src/tools/index.ts`: mount prediction and alarm routes.
- Modify `package.json`: add `job:refit-predictions` and `job:evaluate-alarms` scripts.
- Test `tests/domain/predictions.test.ts`: prediction regression, tau fallback, tau learning, cache upsert.
- Test `tests/domain/alarms.test.ts`: rule validation, daily fire suppression, threshold checks, delivery ordering.
- Test `tests/tools/predictions.test.ts`: authenticated prediction tool routes.
- Test `tests/tools/alarms.test.ts`: authenticated alarm rule tool routes.
